# Button Component

## Overview

The Button component is a versatile UI element that can be used for various user interactions. It supports different sizes, hierarchies, states, and icon configurations, making it suitable for various contexts within your application.

## Usage Guidelines

### When to Use
- Use buttons for actions that trigger a response or navigation, such as "Submit", "Cancel", or "Learn More".
- Prefer buttons over links when the action is a primary task in the user flow.

### When NOT to Use
- Avoid using buttons for non-interactive elements, such as static text or labels.
- Do not use buttons for actions that don't require user confirmation or execution.

## Do's & Don'ts

### Do's
- **Do** use descriptive text within buttons to convey action.
- **Do** ensure the button color contrasts well with the background for visibility.

### Don'ts
- **Don't** use buttons for navigation that could be achieved by links.
- **Don't** use vague labels like "Click here" without context.

## Accessibility

### ARIA
- Use `aria-label` to provide accessible names for buttons that don't contain visible text.
- Ensure buttons have a clear role and state for screen readers.

### Keyboard Navigation
- Buttons should be focusable and operable via the keyboard using the `Enter` or `Space` keys.
- Ensure proper focus management when using buttons in forms or dynamic components.

### Screen Reader
- Ensure all buttons are announced clearly by screen readers.
- Use descriptive labels that convey the action of the button.

## Related Components
- **Link**: For navigation tasks that do not require a confirmation action.
- **IconButton**: For actions that primarily need an icon representation.
- **ToggleButton**: For actions that require switching states or options.
