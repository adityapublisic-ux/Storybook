# Tag Component

## Overview

The Tag component is a versatile UI element used for labeling, categorizing, or indicating status. It supports various sizes, icons, actions, and optional checkboxes, making it adaptable for multiple use cases within your application.

## Usage Guidelines

Use the Tag component when you want to:
- Display a label associated with an item.
- Indicate a status or category.
- Allow users to select items using checkboxes.

Avoid using the Tag component when:
- You require complex interactions (consider using buttons or dropdowns).
- The label does not need to be interactive.

## Properties

| Property  | Type      | Description                                     |
|-----------|-----------|-------------------------------------------------|
| `size`    | `string`  | Size of the tag. Options: `sm`, `md`, `lg`.    |
| `icon`    | `string`  | Icon type. Options: `false`, `Dot`, `Country`, `Avatar`. |
| `action`  | `string`  | Action type. Options: `Text only`, `X close`, `Count`. |
| `checkbox`| `boolean` | Whether the tag has a checkbox.                |

## Do's & Don'ts

### Do's
- **Do** use descriptive text for tags to enhance clarity.
```jsx
<Tag size="sm" icon="dot" action="text" checkbox={false}>
  Active
</Tag>
```

### Don'ts
- **Don't** use vague labels that don't convey meaning.
```jsx
<Tag size="sm" icon="dot" action="text" checkbox={false}>
  Item
</Tag>
```

## Accessibility

- **ARIA Labels**: Use `aria-label` to provide additional context for screen readers.
```jsx
<Tag size="sm" icon="dot" action="text" checkbox={false} aria-label="Active Tag">
  Active
</Tag>
```

- **Keyboard Navigation**: Ensure that users can navigate to and interact with tags using keyboard shortcuts (e.g., Tab to focus, Space to toggle checkboxes).

- **Screen Reader Support**: Make sure the text inside the tag clearly describes its purpose for screen reader users.
