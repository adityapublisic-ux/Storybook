# Buttons

## Overview

The Buttons component is a versatile UI element designed to trigger actions and events. It comes in different styles, sizes, and states, enabling developers to implement various button functionalities across their applications.

## Usage Guidelines

### When to Use
- Use buttons to initiate actions (e.g., submitting a form, opening a modal).
- Opt for primary buttons for primary actions and secondary buttons for secondary actions.
- Utilize disabled buttons to indicate that an action cannot be performed at that moment.

### When NOT to Use
- Avoid using buttons for non-interactive elements (e.g., labels, static text).
- Do not use buttons for navigation when links are more appropriate.

## Do's & Don'ts

### Do's
- **Do** use primary buttons for actions that need user attention.
- **Do** ensure buttons have sufficient contrast against their background.

### Don'ts
- **Don't** style buttons with non-standard styles that confuse users.
- **Don't** use buttons for elements that don't perform actions (e.g., headings).

## Accessibility

- **ARIA Attributes**: Use `aria-disabled="true"` for disabled buttons to inform assistive technologies that the button is not interactive.
- **Keyboard Navigation**: Ensure buttons are focusable and can be activated via the keyboard (typically Enter or Space).
- **Screen Readers**: Provide meaningful text for buttons to convey their purpose. Avoid using only icons without text.

- [Links](#): Use links for navigation instead of buttons when appropriate.
- [Icons](#): Combine buttons with icons for enhanced visual appeal.
